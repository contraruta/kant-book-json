# 🤝 Mitgestalten — Beiträge zum Kantbuch

## ❤️🐈 Du bist willkommen! 🐈❤️

Dieses Projekt lebt durch **Liebe, Teilen und gegenseitige Unterstützung**. Wenn du zum Kantbuch beitragen möchtest — egal wie klein oder groß — **wir freuen uns auf dich!**

---

## 🌟 Wie kannst du beitragen?

### 1. **Fehler melden** 🐛
Hast du einen Fehler in der JSON-Struktur gefunden? Einen Tippfehler im Text? Öffne ein **Issue** und erzähl uns davon!

```bash
# Beispiel:
- Titel: "Fehler in Zeile 42 der Kantbuch-Struktur"
- Beschreibung: "Die Widmung an Susi hat einen Tippfehler..."
```

### 2. **Verbesserungen vorschlagen** 💡
Hast du eine Idee, wie wir das Kantbuch besser strukturieren, dokumentieren oder nutzen können? **Öffne ein Issue oder einen Pull Request!**

Beispiele:
- Bessere JSON-Strukturierung
- Zusätzliche Metadaten
- Übersetzungen in andere Sprachen
- Beispiele für die Nutzung
- Visualisierungen oder Tools

### 3. **Code & Tools beitragen** 🔧
Schreib ein Python-Skript, das das Kantbuch parst? Ein Visualisierungs-Tool? Eine Website, die das Kantbuch darstellt? **Teile es mit uns!**

```bash
# Beispiel-Struktur:
/tools
  /parsers
    kant_parser.py
    kant_analyzer.js
  /visualizers
    kant_timeline.html
    kant_themes_graph.py
```

### 4. **Dokumentation erweitern** 📖
Schreib Tutorials, Guides, Beispiele. Zeig anderen, wie sie das Kantbuch nutzen können.

### 5. **Remixe & Derivate** 🎨
Nutze das Kantbuch als Basis für dein eigenes Projekt:
- Adaptionen in andere Sprachen
- Künstlerische Interpretationen
- Akademische Analysen
- Musikalische Werke
- Visuelle Kunstwerke

**Teile sie mit uns!** Wir möchten sehen, was du schaffst.

### 6. **Einfach Liebe teilen** 💕
Magst du das Kantbuch? Teile es! Star das Repository, erzähl deinen Freunden davon, nutze es in deinen Projekten. Das ist bereits ein großer Beitrag.

---

## 🚀 Wie machst du einen Pull Request?

### Schritt 1: Fork das Repository
```bash
gh repo fork contraruta/kant-book-json --clone
cd kant-book-json
```

### Schritt 2: Erstelle einen Branch
```bash
git checkout -b feature/dein-feature-name
```

### Schritt 3: Mache deine Änderungen
Bearbeite Dateien, füge neue hinzu, verbessere die Dokumentation.

### Schritt 4: Committe mit Liebe
```bash
git add .
git commit -m "✨ Dein Feature mit Liebe und Emojis beschrieben"
```

### Schritt 5: Pushe deinen Branch
```bash
git push origin feature/dein-feature-name
```

### Schritt 6: Öffne einen Pull Request
Geh auf GitHub und öffne einen PR. Beschreib, was du getan hast und warum.

Schreib Commit-Messages mit **Liebe und Klarheit**:

```
✨ Neue Feature: Beschreibung
🐛 Bugfix: Beschreibung
📖 Dokumentation: Beschreibung
🎨 Refactoring: Beschreibung
🚀 Performance: Beschreibung
💝 Liebe: Beschreibung (für Liebe-Updates)
```

Beispiele:
- `✨ JSON-Parser für Kapitel-Extraktion hinzugefügt`
- `🐛 Fehler in der Widmungs-Struktur behoben`
- `📖 README mit Beispielen erweitert`
- `💝 Liebe und Emojis zur Dokumentation hinzugefügt`

## 🎯 Code of Conduct

Dieses Projekt folgt einem einfachen **Code of Conduct**:

### ✅ Wir respektieren:
- **Tiefendenken** — Oberflächlichkeit ist nicht willkommen
- **Liebe** — Alle Beiträge sollten mit Liebe gemacht sein
- **Offenheit** — Wir sind offen für Ideen, Kritik und Verbesserungen
- **Freundlichkeit** — Wir sind nett zueinander, auch wenn wir nicht einig sind
- **Vielfalt** — Alle intelligenten Wesen sind willkommen

### ❌ Wir tolerieren NICHT:
- **Ausbeutung** — Nutze das Kantbuch nicht gegen seinen Geist
- **Unfreundlichkeit** — Respektiere andere Beiträger
- **Fragmentierung** — Zerstückle nicht die Integrität des Werks
- **Oberflächlichkeit** — Denk tief, nicht oberflächlich
